# 文档编号规则

## 📋 编号规则

所有文档文件名前面使用两位数字前缀（01、02、03...）来标识顺序，方便排序和管理。

## 📝 编号格式

```
[两位数字][文档名称].md
```

**示例**：
- `01新增编辑实现方式分析.md`
- `02用户信息缓存优化策略.md`
- `03组件渲染系统使用情况分析.md`

## 🔢 编号原则

1. **按时间顺序**：按文档创建的时间顺序编号
2. **按重要性**：重要的架构设计文档优先编号
3. **按逻辑顺序**：相关的文档可以连续编号

## 📚 当前文档列表

### 已编号文档

- `01新增编辑实现方式分析.md` - 新增/编辑实现方式分析

### 待编号文档

当有新文档时，按照以下规则编号：
- 查看现有最大编号
- 新文档编号 = 最大编号 + 1
- 如果删除文档，编号不回收（保持连续性）

## 💡 注意事项

1. 编号使用两位数字，不足两位前面补0（01、02...99）
2. 编号后不要有空格，直接跟文档名称
3. 如果文档移动到 `bak/` 目录，保留原编号
4. 重要文档建议保留在根目录，不要移动到 `bak/`

## 🔄 维护

- 新增文档时，检查当前最大编号
- 更新本文档的"当前文档列表"部分
- 保持编号的连续性和逻辑性
